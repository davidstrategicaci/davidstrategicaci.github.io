NSG GPKG and OSM Route Analysis¶

The following notebook contains working samples of route analysis using NSG and OSM data.

The follow code was developed using the Anaconda deployment python 3.7.

Data Processing Flow¶

The two data used in this working sample are:

  • OSM, Collected through OSMnx
  • An NSG Routing dataset (dervied from OSM, translated in the NSG Feature Dataset Dictionary (NFDD)

We'll use two main libraries to perform route analysis on our datasets:

  • [OSMnx] (https://github.com/gboeing/osmnx)
  • [Dijkstar] (https://pypi.org/project/Dijkstar/)

The NSG data retains the original Node and Edge id from the OSMnx extracted area, however the attributes were translated to fit the NFDD coded values for.

The Following SQL was used to translate from OSM attributes to NFDD compliant attributes:

Set NSG Graph Variables¶

The NSG data is formated to GeoPackage where:

  • Feature Table Edges contains the edge and edge attributes for cost/weight analysis
  • Feature Table Nodes contains the nodes in that make up the graph
  • Attribute Table Network associates the connectivity between Nodes and Edges

Using the SQLite3 Library, we connect to the GeoPackage. The we create a that is read into our Dijkstar Graph class that contains the follwing:

  • the start node (from) and end node (towards) for each edge
  • the cost to travel Start to End
  • the cost to travel End to Start
    • if the edge has a oneway restriction, the it only applies to the start-end direction. A penalty is then applied to the end-start direction to elimiated the possibitility to traverse on that edge in the end-start direction

The table in then inputed into the the graph object

The GPKG is read in to a Geodataframe for visualization

# Set the Proj environment variable for projections 
os.environ['PROJ_LIB']=r"C:\ProgramData\Anaconda3\Library\share"  


GeoPackageFile = "dc_nsg.gpkg"
databaseConnection = sqlite3.connect(GeoPackageFile)
cursor = databaseConnection.cursor()

graph_query = """
select e.edge, startn.node as start_node,endn.node as end_node, e.lzn as cost
from edges e
join (select edge, node from network where sequence_id =0) startn on startn.edge = e.edge
left join (select edge, node from network where sequence_id =1) endn on endn.edge = e.edge

UNION 
--backward
select e.edge, endn.node as start_node,startn.node as end_node, 
	CASE
		when one=1 then 10000000000.0
		else lzn
	END as cost
from edges e
join (select edge, node from network where sequence_id =0) startn on startn.edge = e.edge
left join (select edge, node from network where sequence_id =1) endn on endn.edge = e.edge
"""

cursor.execute(graph_query)

# Network Graph
nsgGraph = Graph()

    
for row in cursor.fetchall():
    nsgGraph.add_edge(row[1],row[2], {'edge_id':row[0],'cost':row[3]})
    
    
# Route Function helpers
nsg_cost_func = lambda u, v, e, prev_e: e['cost']

# for visualization
nsgEdges = gpd.read_file(r'dc_nsg.gpkg', layer='edges')
nsgNodes = gpd.read_file(r'dc_nsg.gpkg', layer='nodes')
